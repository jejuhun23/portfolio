안녕하세요. 백엔드 개발자를 희망하는 김재헌입니다. 저의 가장 큰 강점은 꾸준함을 통해 반드시 결과를 만들어낸다는 것입니다. '카체블(카페인 체내 블랙박스)' 프로젝트에서는 중고등학생의 카페인 과다섭취 문제를 해결하기 위한 서비스를 개발하며 프론트엔드 UI 구현을 담당했습니다. 개발 철학: AI 도구를 활용하되 의존하지 않고, 코드의 본질을 이해하며 개발합니다. 팀 프로젝트에서 적극적인 소통과 협업을 통해 최적의 결과를 도출하고, 사용자의 실제 문제를 해결하는 가치 있는 서비스를 만드는 것을 목표로 합니다.

프로젝트를 통해 배운 점

협업과 의사소통

팀으로 개발하면서 소통이 정말 중요하다는 걸 느꼈습니다. 백엔드팀이랑 API 어떻게 주고받을지 정하고, 디자이너와 화면 구성 상의하는 게 생각보다 시간이 많이 걸렸어요. 노션에 문서로 정리해두니까 나중에 보기 편하더라고요. Git도 처음엔 어려웠는데, 커밋 메시지 제대로 쓰고 브랜치 나누니까 협업이 훨씬 수월했습니다.

AI 도구 사용법

Cursor랑 GitHub Copilot 쓰면서 AI를 무작정 믿으면 안 되겠다는 걸 배웠어요. AI가 짜준 코드 그대로 쓰니까 나중에 이상한 부분이 생기더라고요. 그래서 지금은 AI 코드를 받으면 꼭 한 줄씩 읽어보고 이해한 다음에 쓰고 있습니다. 편하긴 한데, 결국 내가 이해하고 있어야 나중에 문제가 안 생겨요.

부족한 부분 채우기

프로젝트 하면서 내가 뭘 모르는지 확실히 알게 됐어요. 특히 백엔드팀이랑 API 연결할 때 서버 로직을 이해 못 해서 고생했습니다. 그래서 틈틈이 Node.js랑 Express 공부했어요. 프론트만 하려고 했는데 백엔드도 좀 알아야 제대로 협업이 되더라고요. 모르는 거 인정하고 배우는 게 중요한 것 같아요.

사용자 입장에서 생각하기

타겟이 중고등학생이다 보니 직관적이게 만드는 게 진짜 중요했어요. 처음에는 기능 많이 넣으려고 했는데, 실제로 학생들한테 테스트해보니까 복잡하면 안 쓰더라고요. 그래서 버튼 줄이고 화면 구성도 심플하게 바꿨습니다. 개발자 입장이 아니라 실제 쓰는 사람 입장에서 봐야 한다는 걸 느꼈어요.

3D 액션 서바이벌 게임

Unity 엔진과 C#을 활용하여 개발한 3D 액션 게임입니다. 플레이어는 회전하는 플랫폼 위에서 계속해서 등장하는 몬스터를 처치하며 생존하는 것이 목표입니다. 간단한 원터치 조작으로 누구나 쉽게 플레이할 수 있도록 설계했습니다.

게임 룰 & 조작

  • 자동 회전: 플레이어가 원형 플랫폼을 따라 계속 돌아감
  • 간단한 조작: 마우스 클릭만으로 플레이어 이동
  • 게임 목표: 몬스터를 처치하면서 최대한 오래 살아남기
  • 보스 등장: 점수가 일정 수준 이상 올라가면 보스 출현

몬스터 시스템

  • 일반 몬스터: 플레이어를 계속 쫓아오도록 구현
  • 스폰 시스템: 랜덤한 위치에서 몬스터가 계속 생성됨
  • 보스 몬스터: 일반 몬스터보다 체력 3배 + 점프 공격 가능
  • 충돌 감지: 플레이어와 부딪히면 게임 오버

1
게임 시작

일정 시간마다 랜덤 위치에서 몬스터가 등장합니다

몬스터를 처치하며 점수를 획득하고 생존합니다

4

일정 점수 도달 시 보스 몬스터가 출현합니다

5
난이도 상승

보스 처치 후 더 많은 몬스터와 함께 게임이 계속됩니다

게임 디자인

버전 관리
